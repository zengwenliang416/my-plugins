# 输出格式规范

## 概述

对每一页 PPT，按如下字段输出。可用列表或类似 YAML 的结构，但不要嵌套太深。

## 必填字段

### slide_index

- **类型**: 整数
- **说明**: 页码，从 1 开始
- **示例**: `slide_index: 3`

### slide_type

- **类型**: 字符串枚举
- **可选值**:
  | 类型 | 说明 |
  |------|------|
  | `cover` | 封面页 |
  | `story` | 痛点故事页 |
  | `overview` | 问题总览页 |
  | `pitfall` | 深坑/问题页 |
  | `solution` | 解决方案页 |
  | `structure` | 结构说明页 |
  | `demo` | 演示策略页 |
  | `comparison` | 对比页 |
  | `summary` | 总结页 |
  | `cta` | 行动呼吁页 |

### title

- **类型**: 字符串
- **说明**: 页面主标题，简短有力
- **要求**: 不超过 15 字为佳
- **示例**: `title: 痛点不痛：评委为什么没感觉？`

### key_points

- **类型**: 字符串数组
- **说明**: 适合直接放在 PPT 上作为 bullet 的短句
- **要求**:
  - 2–4 条
  - 每条不超过 25 字
  - 避免整句长段落
- **示例**:
  ```yaml
  key_points:
    - 用"效率低、体验差"这类空话，很难打动评委
    - 缺少具体人物、场景和任务，痛点显得不真实
    - 没有数据支撑，项目价值模糊
  ```

### speaker_notes

- **类型**: 字符串数组
- **说明**: 口语化讲解，供演讲者说，不放到 PPT 上
- **要求**:
  - 3–6 句
  - 口语化、易讲易记
  - 可包含故事、比喻、过渡语
- **示例**:
  ```yaml
  speaker_notes:
    - 先讲一个具体角色，比如"财务部张姐"
    - 描述她月底要核对上千张发票的具体任务
    - 点出错误率、加班、投诉等具体后果
    - 说明：如果只说"效率低"，评委完全没有画面感
  ```

### image_prompt

- **类型**: 字符串
- **说明**: 给 banana-image 用的图片描述
- **要求**: 参见 `visual-rules.md` 中的详细规范
- **示例**:
  ```yaml
  image_prompt: >
    扁平插画风格，一位疲惫的财务人员坐在办公桌前，
    被成堆发票和文件包围，桌上有计算器和电脑屏幕，
    表情焦虑和疲惫，背景是略显压迫感的办公室环境，
    主色调蓝紫和浅灰，线条简洁，科技商务风
  ```

## 可选字段

### subtitle

- **类型**: 字符串
- **说明**: 必要时用一句话补充说明
- **示例**: `subtitle: 只有模糊抱怨，没有真实场景`

### visual_layout_hint

- **类型**: 字符串
- **说明**: 简要说明版式布局
- **常用值**:
  | 布局 | 适用场景 |
  |------|---------|
  | 左文右图 | 标准讲解页 |
  | 上标题+中间三列对比 | 对比分析页 |
  | 流程图 | 步骤说明页 |
  | 中心放射 | 概览总览页 |
  | 左右对比框 | 错误vs正确示例 |
  | 表格 | 数据对比页 |

## 完整示例

```yaml
slide_index: 3
slide_type: pitfall
title: 痛点不痛：评委为什么没感觉？
subtitle: 只有模糊抱怨，没有真实场景
key_points:
  - 用"效率低、体验差"这类空话，很难打动评委
  - 缺少具体人物、场景和任务，痛点显得不真实
  - 没有数据支撑，项目价值模糊
speaker_notes:
  - 先讲一个具体角色，比如"财务部张姐"
  - 描述她月底要核对上千张发票的具体任务
  - 点出错误率、加班、投诉等具体后果
  - 说明：如果只说"效率低"，评委完全没有画面感
visual_layout_hint: 左侧是抽象说法的错误示例，右侧是具体故事化表达的对比框
image_prompt: >
  扁平插画风格，一位疲惫的财务人员坐在办公桌前，
  被成堆发票和文件包围，桌上有计算器和电脑屏幕，
  表情焦虑和疲惫，背景是略显压迫感的办公室环境，
  主色调蓝紫和浅灰，线条简洁，科技商务风
```

## 输出顺序

生成 PPT 时，按以下顺序输出：

1. **整体结构概览**（简要列出所有页面标题）
2. **逐页详细输出**（按 slide_index 顺序）

### 整体结构概览格式

```markdown
## 整体结构概览

| 页码 | 类型     | 标题                       |
| ---- | -------- | -------------------------- |
| 1    | cover    | AI编程大赛项目汇报避坑指南 |
| 2    | story    | 一个真实的翻车故事         |
| 3    | overview | 两大类问题：内容 vs 表达   |
| ...  | ...      | ...                        |
```

---

## 字段校验规则

| 字段          | 校验规则                             |
| ------------- | ------------------------------------ |
| slide_index   | 必须连续，从 1 开始                  |
| slide_type    | 必须是枚举值之一                     |
| title         | 非空，建议不超过 15 字               |
| key_points    | 2-4 条，每条不超过 25 字             |
| speaker_notes | 3-6 句，口语化                       |
| image_prompt  | 必须包含风格、场景、人物、情绪、色调 |
