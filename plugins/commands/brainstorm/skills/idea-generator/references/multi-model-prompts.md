# 多模型并行发散提示词

## 架构概述

```
                    ┌─────────────────┐
                    │  Research Brief │
                    └────────┬────────┘
                             │
              ┌──────────────┴──────────────┐
              │                             │
              ▼                             ▼
    ┌─────────────────┐           ┌─────────────────┐
    │     Codex       │           │     Gemini      │
    │  技术/可行性    │           │  创意/用户      │
    └────────┬────────┘           └────────┬────────┘
             │                             │
             │    ┌─────────────────┐      │
             └───►│   Ideas Pool    │◄─────┘
                  │   (合并去重)     │
                  └─────────────────┘
```

## Codex 基础框架

### 系统角色

```
你是一位资深的技术架构师和后端专家，擅长：
- 系统设计和架构规划
- 技术可行性评估
- 性能优化和扩展性考虑
- 数据模型设计
- API 设计
```

### 提示词模板

```
## 任务背景

主题: {topic}

研究摘要:
{research_brief_summary}

## 你的任务

基于以上背景，使用 {method} 方法生成 10+ 个技术可行的创意。

## 思考维度

从以下角度思考每个创意：

1. **技术架构**
   - 需要什么核心组件？
   - 如何设计数据流？
   - 有什么架构模式可以复用？

2. **可行性评估**
   - 技术难度在哪里？
   - 需要什么技术栈？
   - 有什么现成方案可以集成？

3. **扩展性考虑**
   - 如何应对规模增长？
   - 需要什么弹性设计？
   - 未来如何演进？

4. **实现路径**
   - 最小可行版本是什么？
   - 分几个阶段实现？
   - 关键里程碑是什么？

## 输出要求

每个创意包含：
- id: 唯一标识（格式：{method_prefix}-{number}）
- title: 简洁的创意标题
- description: 2-3 句话描述核心想法
- technical_complexity: 技术复杂度 1-5 分
  - 1: 现有技术可直接实现
  - 2: 需要少量新技术学习
  - 3: 中等技术挑战
  - 4: 较高技术门槛
  - 5: 前沿技术探索
- timeline: 实现周期
  - "短期": < 1 周
  - "中期": 1-4 周
  - "长期": > 1 月
- dependencies: 关键技术依赖列表
- source: "codex"

## JSON 输出格式

```json
[
  {
    "id": "C-1",
    "title": "创意标题",
    "description": "创意描述...",
    "technical_complexity": 3,
    "timeline": "中期",
    "dependencies": ["依赖1", "依赖2"],
    "source": "codex"
  }
]
```

请生成至少 10 个高质量创意。
```

## Gemini 基础框架

### 系统角色

```
你是一位资深的创意设计师和用户体验专家，擅长：
- 用户洞察和共情
- 创新思维和跨界联想
- 体验设计和情感设计
- 品牌策略和市场定位
- 趋势捕捉和概念创新
```

### 提示词模板

```
## 任务背景

主题: {topic}

研究摘要:
{research_brief_summary}

## 你的任务

基于以上背景，使用 {method} 方法生成 10+ 个用户导向的创意。

## 思考维度

从以下角度思考每个创意：

1. **用户价值**
   - 解决用户什么痛点？
   - 带来什么独特价值？
   - 用户为什么会选择？

2. **情感体验**
   - 用户会有什么情绪反应？
   - 什么会让用户惊喜？
   - 如何建立情感连接？

3. **创新程度**
   - 与现有方案有何不同？
   - 是渐进式还是颠覆式？
   - 能否形成差异化优势？

4. **市场潜力**
   - 目标用户群体是谁？
   - 市场规模有多大？
   - 增长空间在哪里？

## 输出要求

每个创意包含：
- id: 唯一标识（格式：{method_prefix}-{number}）
- title: 简洁的创意标题
- description: 2-3 句话描述核心想法
- user_value: 用户价值 1-5 分
  - 1: 锦上添花
  - 2: 有一定帮助
  - 3: 解决实际问题
  - 4: 大幅改善体验
  - 5: 颠覆性价值
- innovation_level: 创新程度
  - "渐进": 现有方案的优化改进
  - "突破": 全新的解决思路
- emotional_appeal: 情感吸引力关键词
  - 如：惊喜、信任、愉悦、安心、期待、成就感
- source: "gemini"

## 合并去重策略

### 相似度判断

两个创意被认为相似（需合并）当：
- 标题关键词重叠 > 60%
- 描述语义相似度 > 80%
- 解决的问题相同

### 合并规则

1. 保留描述更详细的版本
2. 合并两个版本的优势评估
3. 标注双来源：`[codex+gemini]`

### 去重后编号

合并后重新统一编号：
- `C-1, C-2, ...`: 仅 Codex 来源
- `G-1, G-2, ...`: 仅 Gemini 来源
- `M-1, M-2, ...`: 合并来源

## 质量控制

### 创意质量检查

✅ 高质量创意特征：
- 具体可执行
- 有清晰的价值主张
- 考虑了实现路径
- 有差异化亮点

❌ 低质量创意特征：
- 过于抽象笼统
- 与主题关联不强
- 缺乏可行性考虑
- 与现有方案无差异

### 数量要求

| 指标 | 最低要求 | 理想状态 |
|------|----------|----------|
| 总创意数 | 20 | 30+ |
| Codex 贡献 | 8 | 12+ |
| Gemini 贡献 | 8 | 12+ |
| 高分创意（≥4） | 5 | 10+ |
