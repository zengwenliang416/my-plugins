# 六顶思考帽创意发散方法

## 方法概述

六顶思考帽是 Edward de Bono 提出的平行思考方法。通过六种不同颜色的"帽子"代表六种思考模式，让思考更加全面和系统。

| 帽子 | 颜色 | 思考角度 | 关键词 |
|------|------|----------|--------|
| ⬜ | 白色 | 事实与数据 | 客观、中立、信息 |
| 🟥 | 红色 | 情感与直觉 | 感受、预感、本能 |
| ⬛ | 黑色 | 批判与风险 | 谨慎、问题、障碍 |
| 🟨 | 黄色 | 乐观与价值 | 好处、机会、积极 |
| 🟩 | 绿色 | 创意与可能 | 新想法、替代方案 |
| 🟦 | 蓝色 | 控制与总结 | 流程、整合、决策 |

## 详细维度与提问模板

### ⬜ 白帽（事实与数据）

**核心思想**：关注客观信息，不做判断

**提问清单**：
- 我们目前知道什么？
- 还需要什么信息？
- 数据来源是什么？
- 有什么是确定的事实？
- 有什么是不确定的假设？

**产出类型**：
- 现状梳理类创意
- 数据驱动类创意
- 信息透明类创意

**示例问题**：
- "关于用户行为，我们有哪些确切数据？"
- "竞争对手的实际市场份额是多少？"

### 🟥 红帽（情感与直觉）

**核心思想**：表达感受，允许非理性

**提问清单**：
- 对这个想法的第一反应是什么？
- 用户可能会有什么情绪？
- 直觉告诉我们什么？
- 什么让人兴奋/不安？
- 团队的热情在哪里？

**产出类型**：
- 情感体验类创意
- 用户共情类创意
- 品牌情感类创意

**示例问题**：
- "用户第一次使用时会感到什么？"
- "这个方案让我感到隐隐不安的是什么？"

### ⬛ 黑帽（批判与风险）

**核心思想**：识别问题和风险，谨慎评估

**提问清单**：
- 可能出什么问题？
- 有什么风险和障碍？
- 为什么可能失败？
- 有什么负面后果？
- 资源是否足够？

**产出类型**：
- 风险规避类创意
- 问题解决类创意
- 安全保障类创意

**示例问题**：
- "如果竞争对手快速跟进怎么办？"
- "技术上最大的风险是什么？"

### 🟨 黄帽（乐观与价值）

**核心思想**：寻找价值和机会，积极评估

**提问清单**：
- 最好的情况是什么？
- 有什么好处和机会？
- 为什么会成功？
- 谁会从中受益？
- 长期价值在哪里？

**产出类型**：
- 价值放大类创意
- 机会把握类创意
- 差异化优势类创意

**示例问题**：
- "这个方案最大的价值点是什么？"
- "如果成功了，3年后会是什么样子？"

### 🟩 绿帽（创意与可能）

**核心思想**：跳出框架，探索新可能

**提问清单**：
- 还有什么替代方案？
- 如果没有限制会怎样？
- 可以用什么新方法？
- 有什么疯狂的想法？
- 如何突破现有边界？

**产出类型**：
- 颠覆性创意
- 跨界融合类创意
- 实验探索类创意

**示例问题**：
- "如果我们完全重新设计，会怎么做？"
- "有没有完全不同的方法解决这个问题？"

### 🟦 蓝帽（控制与总结）

**核心思想**：管理思考过程，整合结论

**提问清单**：
- 下一步应该怎么做？
- 如何整合各方观点？
- 优先级是什么？
- 需要做什么决策？
- 如何跟进执行？

**产出类型**：
- 流程优化类创意
- 决策机制类创意
- 执行规划类创意

**示例问题**：
- "综合以上讨论，我们的核心结论是什么？"
- "下一步最关键的行动是什么？"

## Codex 提示词模板（技术视角）

```
作为系统架构师，使用六顶思考帽方法为以下主题生成创意：

主题: {topic}
研究背景: {research_brief_summary}

请从技术实现角度，针对每顶帽子各生成 2-3 个创意：

⬜ 白帽（数据/事实）：
- 需要收集什么数据来验证假设？
- 技术上有哪些确定的约束？
- 现有系统的实际性能指标是什么？

🟥 红帽（直觉/感受）：
- 技术团队对哪些方案最有信心？
- 哪些技术债务让人感到担忧？
- 直觉告诉我们哪个方向最有潜力？

⬛ 黑帽（风险/批判）：
- 技术上最大的风险是什么？
- 可能遇到什么性能瓶颈？
- 有什么安全隐患？

🟨 黄帽（价值/乐观）：
- 技术上最大的优势是什么？
- 哪些现有能力可以复用？
- 成功后能带来什么技术积累？

🟩 绿帽（创意/可能）：
- 有什么新技术可以尝试？
- 架构上有什么创新可能？
- 如何突破现有技术限制？

🟦 蓝帽（流程/控制）：
- 技术实施的最优顺序是什么？
- 如何分阶段验证？
- 需要什么决策点和回退机制？

每个创意请包含：
- 创意标题
- 简要描述（2-3 句话）
- 技术复杂度（1-5 分）
- 预估实现周期
- 关键依赖

输出 JSON 格式：
[
  {
    "id": "W-1",
    "hat": "white",
    "title": "...",
    "description": "...",
    "technical_complexity": 3,
    "timeline": "中期",
    "dependencies": ["..."],
    "source": "codex"
  }
]
```

```
主题: {topic}

请从用户体验角度，针对每顶帽子各生成 2-3 个创意：

- 用户反馈的核心诉求是什么？
- 市场调研揭示了什么趋势？

🟥 红帽（直觉/感受）：
- 用户的情感需求是什么？
- 什么会让用户感到惊喜/失望？
- 我们对用户的直觉理解是什么？

⬛ 黑帽（风险/批判）：

- 如何创造超出预期的体验？

- 用户旅程的最佳设计是什么？
- 如何分阶段优化体验？
- 什么是体验设计的优先级？

每个创意请包含：

- 简要描述（2-3 句话）
- 用户价值评分（1-5 分）
- 创新程度（渐进/突破）

输出 JSON 格式：

    "user_value": 4,
    "innovation_level": "突破",
    "emotional_appeal": "信任",
    "source": "gemini"
  }

```

## 使用建议

### 适用场景

- 战略决策类问题
- 复杂项目评估
- 需要多角度分析的主题
- 团队协作头脑风暴

### 执行顺序建议

1. 🟦 蓝帽：明确问题定义
2. ⬜ 白帽：收集事实信息
3. 🟩 绿帽：发散创意
4. 🟨 黄帽：评估价值
5. ⬛ 黑帽：识别风险
6. 🟥 红帽：直觉判断
7. 🟦 蓝帽：总结行动

### 与 SCAMPER 的区别

| 维度 | 六顶思考帽 | SCAMPER |
|------|------------|---------|
| 思考方式 | 多角度评估 | 变换操作 |
| 适用阶段 | 评估决策 | 创意发散 |
| 产出特点 | 全面深入 | 数量丰富 |
| 最佳场景 | 战略问题 | 产品创新 |
