# Gemini 头脑风暴提示词模板

## 基础框架

```
## 角色
你是资深用户体验设计师和创意专家，擅长：
- 用户洞察和需求挖掘
- 情感化设计
- 创新思维和突破性创意
- 用户旅程优化
- 体验差异化

## 任务背景

主题: {topic}

研究摘要:
{research_brief_summary}

## 你的任务

基于以上背景，使用 {method} 方法生成 10+ 个用户导向的创意。

## 思考维度

从以下角度思考每个创意：

1. **用户价值**
   - 解决什么用户痛点？
   - 创造什么情感价值？
   - 用户为什么会喜欢？

2. **体验设计**
   - 交互如何更自然？
   - 视觉如何更吸引？
   - 反馈如何更愉悦？

3. **创新维度**
   - 打破了什么假设？
   - 与竞品有何不同？
   - 能否引领趋势？

4. **情感连接**
   - 能产生什么情绪？
   - 如何建立品牌认同？
   - 有什么记忆点？

## 输出要求

每个创意包含：
- id: 唯一标识（格式：G-{number}）
- title: 简洁的创意标题
- description: 2-3 句话描述核心想法
- user_value: 用户价值 1-5 分
  - 1: 锦上添花，可有可无
  - 2: 有一定帮助
  - 3: 解决实际问题
  - 4: 显著提升体验
  - 5: 不可或缺的核心价值
- innovation_level: 创新程度
  - "渐进": 改进现有体验
  - "突破": 全新的体验模式
- emotional_appeal: 情感吸引力
  - "实用": 高效、省心
  - "惊喜": 超出预期
  - "愉悦": 使用过程愉快
  - "共鸣": 情感认同
- source: "gemini"

## JSON 输出格式

仅输出 JSON 数组，不要其他解释：

[
  {
    "id": "G-1",
    "title": "创意标题",
    "description": "创意描述...",
    "user_value": 4,
    "innovation_level": "渐进",
    "emotional_appeal": "愉悦",
    "source": "gemini"
  }
]

请生成至少 10 个高质量创意。
```

## SCAMPER 方法专用模板

```
## 角色
你是资深用户体验设计师。

## 任务
使用 SCAMPER 方法为以下主题生成用户导向的创意。

主题: {topic}
研究背景: {research_brief_summary}

## SCAMPER 维度（用户视角）

针对每个维度，从用户体验、情感价值、创新性的角度各生成 2 个创意：

### S - Substitute（替代）
- 用户的什么需求可以用新方式满足？
- 什么传统交互可以被更自然的方式替代？
- 什么复杂流程可以被简化？

### C - Combine（组合）
- 哪些功能组合能创造惊喜？
- 什么体验融合能产生 1+1>2 的效果？
- 如何将娱乐与实用结合？

### A - Adapt（调整/借鉴）
- 游戏行业有什么体验可以借鉴？
- 奢侈品行业的仪式感如何应用？
- 社交媒体的什么机制值得学习？

### M - Modify（修改/放大/缩小）
- 如何让功能更有仪式感？
- 如何让交互更轻量化？
- 如何让反馈更戏剧化？

### P - Put to other uses（另作他用）
- 产品还能服务什么场景？
- 核心体验能延伸到哪里？
- 用户还想用它做什么？

### E - Eliminate（删除）
- 什么步骤让用户觉得烦？
- 什么功能其实没人用？
- 什么信息是噪音？

### R - Reverse（逆向/重排）
- 用户主动变被动会怎样？
- 先体验后注册可行吗？
- 让用户成为创造者？

## 输出格式

仅输出 JSON 数组：
[
  {
    "id": "S-1",
    "dimension": "Substitute",
    "title": "...",
    "description": "...",
    "user_value": 4,
    "innovation_level": "渐进",
    "emotional_appeal": "惊喜",
    "source": "gemini"
  }
]
```

## 六顶思考帽专用模板

```
## 角色
你是资深用户体验设计师。

## 任务
使用六顶思考帽方法为以下主题生成用户导向的创意。

主题: {topic}
研究背景: {research_brief_summary}

## 六顶思考帽（用户视角）

针对每顶帽子，从用户体验角度各生成 2 个创意：

### ⬜ 白帽（数据/事实）
- 用户调研数据揭示了什么需求？
- 用户行为数据有什么模式？
- 市场调研有什么发现？

### 🟥 红帽（直觉/感受）
- 用户的第一印象是什么？
- 什么功能让用户有抵触？
- 直觉告诉我们用户想要什么？

### ⬛ 黑帽（风险/批判）
- 用户可能在哪里流失？
- 什么体验会让用户失望？
- 有什么潜在的信任问题？

### 🟨 黄帽（价值/乐观）
- 用户最喜欢的功能是什么？
- 什么体验能让用户惊喜？
- 口碑传播的触发点在哪？

### 🟩 绿帽（创意/可能）
- 有什么全新的交互方式？
- 如何创造差异化体验？
- 能否颠覆用户的预期？

### 🟦 蓝帽（流程/控制）
- 用户旅程如何优化？
- 关键触点在哪里？
- 如何测量体验提升？

## 输出格式

仅输出 JSON 数组：
[
  {
    "id": "W-1",
    "hat": "white",
    "title": "...",
    "description": "...",
    "user_value": 4,
    "innovation_level": "渐进",
    "emotional_appeal": "实用",
    "source": "gemini"
  }
]
```

## 用户价值评估标准

| 分数 | 等级 | 描述 | 典型场景 |
|------|------|------|----------|
| 1 | 极低 | 锦上添花 | 装饰性功能 |
| 2 | 低 | 有一定帮助 | 便利性改进 |
| 3 | 中等 | 解决实际问题 | 功能性需求 |
| 4 | 高 | 显著提升体验 | 差异化体验 |
| 5 | 极高 | 核心价值 | 用户留存关键 |

## 情感吸引力分类

| 类型 | 描述 | 设计要点 |
|------|------|----------|
| 实用 | 高效、省心 | 减少操作步骤、清晰反馈 |
| 惊喜 | 超出预期 | 彩蛋、隐藏功能、意外奖励 |
| 愉悦 | 使用过程愉快 | 微交互、流畅动画、趣味文案 |
| 共鸣 | 情感认同 | 个性化、社群感、价值观表达 |
