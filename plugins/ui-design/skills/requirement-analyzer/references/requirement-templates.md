# UI/UX Requirement Templates Reference

需求分析模板和设计要素提取指南。

---

## 1. 需求分析框架

### 1.1 分析维度

| 维度 | 分析内容 | 输出 |
|------|----------|------|
| 产品类型 | 网站/App/Dashboard/工具 | 类型标签 |
| 目标用户 | 年龄/职业/技术水平 | 用户画像 |
| 核心功能 | 主要功能列表 | 功能清单 |
| 设计偏好 | 风格/色调/品牌 | 设计约束 |
| 技术约束 | 框架/浏览器/设备 | 技术边界 |

### 1.2 用户画像模板

```markdown
## 用户画像

### 主要用户群

**人口特征**:
- 年龄范围: [范围]
- 职业背景: [职业]
- 技术水平: [初级/中级/高级]

**使用场景**:
- 主要场景: [场景描述]
- 使用频率: [高/中/低]
- 使用设备: [桌面/移动/平板]

**核心需求**:
- [需求 1]
- [需求 2]

**痛点**:
- [痛点 1]
- [痛点 2]
```

## 5. 可访问性要求

### 5.1 WCAG 等级

```markdown
## 可访问性

### 目标等级

- [ ] WCAG 2.1 A
- [ ] WCAG 2.1 AA (推荐)
- [ ] WCAG 2.1 AAA

### 具体要求

**感知性**:
- [ ] 文本替代（图片 alt）
- [ ] 颜色对比度 ≥4.5:1
- [ ] 可调整文本大小

**可操作性**:
- [ ] 键盘可访问
- [ ] 足够的点击区域 (≥44px)
- [ ] 无时间限制

**可理解性**:
- [ ] 清晰的标签
- [ ] 一致的导航
- [ ] 错误提示友好

**健壮性**:
- [ ] 语义化 HTML
- [ ] ARIA 属性
```

## 7. 需求澄清问题

### 7.1 必问问题

当用户需求模糊时，使用 AskUserQuestion 询问：

```
1. 产品类型
   "这是什么类型的产品？"
   选项: 网站, Web App, 移动 App, Dashboard, 其他

2. 目标用户
   "主要用户群体是？"
   选项: 消费者, 企业用户, 开发者, 内部员工

3. 设计偏好
   "偏好什么设计风格？"
   选项: 极简, 现代专业, 活泼友好, 其他

4. 技术栈
   "使用什么前端框架？"
   选项: React, Vue, Angular, 其他
```

### 7.2 可选问题

```
- 是否有品牌色？
- 是否有参考设计？
- 是否需要深色模式？
- 响应式优先级？
```
