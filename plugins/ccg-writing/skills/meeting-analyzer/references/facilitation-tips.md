# 会议引导技巧

## 会前准备

### 确定会议必要性

```
需要开会吗？

问自己：
├── 这件事能通过邮件/消息解决吗？
│   └── 是 → 不开会，发消息
├── 需要实时讨论/决策吗？
│   └── 否 → 不开会，异步沟通
├── 需要多人同时参与吗？
│   └── 否 → 不开会，一对一沟通
└── 以上都是"否" → 需要开会
```

### 会议准备清单

**主持人准备**：

- [ ] 明确会议目标（可衡量的产出）
- [ ] 制定详细议程（含时间分配）
- [ ] 确定必要参与者
- [ ] 提前发送会议材料
- [ ] 预定会议室/测试会议链接
- [ ] 准备白板/协作工具

**参与者准备**：

- [ ] 阅读会前材料
- [ ] 准备需要分享的内容
- [ ] 准备问题或观点
- [ ] 确保设备/网络正常

## 会议开场

### 开场框架 (2-3 分钟)

```markdown
1. 欢迎与签到
   "感谢大家参加，我们先确认一下参与者..."

2. 说明目标
   "今天会议的目标是 [具体目标]，
   结束时我们需要 [具体产出]"

3. 介绍议程
   "我们有 [X] 分钟，议程如下..."

4. 设定规则
   "请保持手机静音，有问题可以直接打断..."

5. 时间承诺
   "我们会准时在 [时间] 结束"
```

### 破冰技巧

**适用场景**：

- 团队新成员加入
- 跨部门首次协作
- 气氛比较紧张时

**方法**：

1. **一句话自我介绍**: 姓名 + 角色 + 对本次会议的期待
2. **快速问答**: "用一个词形容你现在的状态"
3. **相关话题**: "最近在这个领域遇到什么挑战？"

## 讨论引导

### 提问技巧

**开放式问题**（激发讨论）：

- "大家对这个方案有什么看法？"
- "还有哪些我们可能忽略的因素？"
- "如果资源充足，你会怎么做？"

**封闭式问题**（推进决策）：

- "我们是否同意采用方案 A？"
- "截止日期定在周五，有问题吗？"
- "需要更多时间讨论吗？"

**追问技巧**：

- "能具体说说吗？"
- "你能举个例子吗？"
- "为什么你这样认为？"
- "这对我们意味着什么？"

### 鼓励参与

**针对沉默者**：

```
"[姓名]，你在这个领域很有经验，你怎么看？"
"我们还没听到 [姓名] 的意见，你有什么想法？"
```

**针对主导者**：

```
"谢谢 [姓名] 的分享，我们听听其他人的看法"
"这是很好的观点，我们也想了解不同的视角"
```

**针对离题者**：

```
"这是个好话题，但可能超出今天的范围，我们记下来另找时间讨论"
"让我们先聚焦今天的议题，这个问题我们放到停车场"
```

### 停车场技术

当讨论跑题时：

1. 准备一个"停车场"（白板区域/文档）
2. 记录跑题但有价值的话题
3. 承诺后续处理
4. 拉回正轨

```
┌─────────────────────────────────┐
│           停车场               │
├─────────────────────────────────┤
│ • 数据库选型讨论 → 周三技术会  │
│ • 新功能需求 → 产品经理跟进    │
│ • 团建安排 → 行政处理          │
└─────────────────────────────────┘
```

## 时间管理

### 时间控制技巧

**可视化计时**：

- 使用明显的计时器
- 定期提醒剩余时间
- "还有 5 分钟，我们需要开始总结了"

**弹性议程**：

```
议题 A: 15 分钟 (可压缩)
议题 B: 20 分钟 (核心，不可压缩)
议题 C: 10 分钟 (可推迟)
缓冲:   5 分钟
```

**超时处理**：

- 选项 1: 延长会议（需全员同意）
- 选项 2: 后续单独讨论
- 选项 3: 另安排会议
- 选项 4: 异步决策

## 冲突处理

### 识别冲突信号

**言语信号**：

- 声音提高
- 打断他人
- 否定性语言 ("不可能"、"你不懂")
- 沉默/退出讨论

**非言语信号**：

- 身体后退/交叉手臂
- 翻白眼/摇头
- 看手机/走神
- 紧绷的面部表情

### 冲突调解步骤

```
1. 暂停讨论
   "让我们先暂停一下..."

2. 中性复述
   "我听到两个不同的观点..."

3. 寻找共识
   "你们都同意的部分是..."

4. 聚焦目标
   "让我们回到今天的目标..."

5. 提出前进方式
   "我们可以先试试...之后再评估"
```

### 处理困难参与者

| 类型   | 表现     | 应对策略                       |
| ------ | -------- | ------------------------------ |
| 主导型 | 说话太多 | "谢谢分享，我们听听其他人"     |
| 沉默型 | 不发言   | 直接点名，从简单问题开始       |
| 否定型 | 总是反对 | "你的顾虑是什么？有什么建议？" |
| 跑题型 | 偏离主题 | 使用停车场技术                 |
| 迟到型 | 经常迟到 | 私下沟通，会议准时开始         |

## 决策引导

### 决策方法

**共识决策**（需要全员支持）：

```
"我们需要所有人都同意这个方向，
有没有任何无法接受的顾虑？"
```

**多数决策**（快速推进）：

```
"让我们投票决定，
同意的请举手/打1，反对的打2"
```

**授权决策**（明确责任人）：

```
"这个决策由 [姓名] 负责，
大家有什么建议可以现在提出"
```

**分级同意**（了解支持程度）：

```
1 = 完全支持
2 = 支持但有小顾虑
3 = 可以接受
4 = 有保留但不阻止
5 = 无法接受
```

### 达成共识技巧

1. **先同后异**: 先列出共识点，再讨论分歧
2. **利益而非立场**: "你真正需要的是什么？"
3. **拆分问题**: 把大问题拆成小决策
4. **试点方案**: "我们先小范围试试，再决定"
5. **时间压力**: "我们需要在会议结束前决定"

## 会议收尾

### 总结模板 (3-5 分钟)

```markdown
1. 回顾目标
   "今天我们的目标是 [X]..."

2. 总结决议
   "我们决定了以下几点：
   - 决议 1
   - 决议 2"

3. 明确行动项
   "[姓名] 负责 [任务]，在 [日期] 前完成"

4. 确认下一步
   "下次会议将在 [时间]，讨论 [主题]"

5. 收集反馈
   "对今天的会议有什么建议吗？"

6. 结束致谢
   "感谢大家的参与！"
```

### 行动项 SMART 原则

| 要素           | 含义   | 示例            |
| -------------- | ------ | --------------- |
| **S**pecific   | 具体   | "完成 API 文档" |
| **M**easurable | 可衡量 | "覆盖所有端点"  |
| **A**ssignable | 可分配 | "张三负责"      |
| **R**ealistic  | 现实   | "基于现有模板"  |
| **T**ime-bound | 有时限 | "周五下班前"    |

## 远程会议技巧

### 技术准备

- [ ] 提前测试音视频设备
- [ ] 确保网络稳定
- [ ] 准备备用方案（电话拨入）
- [ ] 共享屏幕准备就绪
- [ ] 关闭无关应用通知

### 远程互动

**保持参与**：

- 要求开启摄像头
- 频繁进行互动（提问、投票）
- 使用协作工具（Miro, Figma）
- 每 10-15 分钟变换形式

**处理干扰**：

- 使用静音功能
- 约定发言信号（举手按钮）
- 设置明确的发言顺序

### 混合会议

```
线上参与者     会议室参与者
    ↓               ↓
  ┌─────────────────────┐
  │    清晰的音视频      │
  │    共享的白板/文档   │
  │    平等的发言机会    │
  └─────────────────────┘
```

**原则**：

- 假设所有人都是远程参与
- 会议室设备对准说话人
- 聊天框同步重要信息
- 明确点名邀请发言
