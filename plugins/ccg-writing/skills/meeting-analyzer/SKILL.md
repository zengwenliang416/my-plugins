# Meeting Analyzer - 会议洞察分析助手

分析会议记录和转写文本，提取关键信息、行动项、决策点，识别沟通模式和团队动态。

## 工作流程

```
1. 确定分析目标 → 2. 解析会议内容 → 3. 提取关键信息 → 4. 生成报告
```

## 分析维度

### 1. 内容分析

- **讨论主题**：识别核心议题和时间分配
- **关键决策**：提取决策内容、提出者、共识程度
- **待决事项**：列出需进一步讨论的事项
- **行动项**：任务、负责人、截止日期、优先级

### 2. 参与者分析

- **发言统计**：发言次数、时长、主要贡献
- **互动模式**：主导者、贡献者、挑战者、沉默者
- **发言比例**：可视化参与度分布

### 3. 沟通模式识别

| 模式类型      | 识别要点           |
| ------------- | ------------------ |
| 正向模式 ✅   | 建设性讨论、共识   |
| 需关注模式 ⚠️ | 回避、冲突、信息差 |

### 4. 情感分析

- 整体情绪和变化趋势
- 敏感话题识别
- 团队士气指标

---

## 低效会议特征

| 问题类型 | 特征                             |
| -------- | -------------------------------- |
| 结构问题 | 无议程、频繁跑题、无时间控制     |
| 参与问题 | 少数人主导、大量沉默、无关人员多 |
| 决策问题 | 议而不决、缺乏数据、责任不明确   |

---

## 输出格式

### 快速摘要（30秒版）

```markdown
**一句话总结**: [核心结论]
**关键决策**: [1-2 个最重要决策]
**紧急行动**: [最紧急的 1-2 个任务]
**下一步**: [下次会议/检查点]
```

### 完整报告

使用 `references/meeting-templates.md` 中的模板生成详细分析。
