# 创意技法大全

## 发散思维技法

### 1. 头脑风暴 (Brainstorming)

**基本规则**：

- 不批评任何想法
- 欢迎疯狂的想法
- 追求数量
- 借力发挥，组合改进

**变体**：

- **反向头脑风暴**: 思考如何让问题更糟糕，然后反转
- **角色头脑风暴**: 假设某个名人/专家会如何解决
- **随机词头脑风暴**: 用随机词汇触发联想

### 2. 思维导图 (Mind Mapping)

```
                    ┌─── 子主题 1.1
           ┌─ 主题 1 ─┤
           │        └─── 子主题 1.2
           │
中心主题 ──┼─ 主题 2 ─── 子主题 2.1
           │
           │        ┌─── 子主题 3.1
           └─ 主题 3 ─┤
                    └─── 子主题 3.2
```

**绘制步骤**：

1. 中心放置核心主题
2. 向外延伸主要分支
3. 继续细分子分支
4. 使用颜色和图标区分
5. 寻找分支间的连接

### 3. 强制关联法 (Forced Connections)

将两个看似无关的概念强制关联，寻找创新点。

**步骤**：

1. 定义问题
2. 随机选择一个无关对象/概念
3. 列出该对象的属性
4. 将每个属性与问题关联
5. 发展有潜力的想法

**示例**：

```
问题: 如何改进会议效率？
随机对象: 厨房

厨房属性 → 会议应用
- 食谱 → 会议议程模板
- 计时器 → 严格时间限制
- 调味品 → 适时的"调味"互动
- 冰箱 → 想法"保鲜"存储
```

### 4. 奔驰法 (SCAMPER)

| 字母 | 技法                     | 问题示例                   |
| ---- | ------------------------ | -------------------------- |
| S    | Substitute (替代)        | 可以用什么材料/流程替代？  |
| C    | Combine (结合)           | 可以与什么功能/产品结合？  |
| A    | Adapt (改造)             | 可以借鉴什么现有方案？     |
| M    | Modify (修改)            | 放大/缩小/改变形状会怎样？ |
| P    | Put to other uses (他用) | 还能用在什么场景？         |
| E    | Eliminate (消除)         | 去掉什么可以更简单？       |
| R    | Reverse (逆向)           | 颠倒顺序/角色会怎样？      |

### 5. 类比思维 (Analogical Thinking)

**自然类比**：

- 蜂巢结构 → 高效存储设计
- 蚂蚁协作 → 分布式系统
- 鱼群游动 → 群体智能算法

**跨行业类比**：

- 航空安全检查 → 代码审查流程
- 医院分诊制度 → 需求优先级管理
- 餐厅服务流程 → 用户体验设计

## 收敛思维技法

### 1. 亲和图法 (Affinity Diagram)

**步骤**：

1. 将所有想法写在便签上
2. 安静地分组（不讨论）
3. 为每组命名
4. 识别组间关系
5. 确定优先级

### 2. 点投票法 (Dot Voting)

**规则**：

- 每人 3-5 个投票点
- 可以集中投或分散投
- 最高票数的想法优先讨论

### 3. 决策矩阵 (Decision Matrix)

| 方案 | 可行性 (x2) | 成本 | 收益 (x2) | 风险 | 加权总分 |
| ---- | ----------- | ---- | --------- | ---- | -------- |
| A    | 8 (16)      | 6    | 9 (18)    | 7    | 47       |
| B    | 6 (12)      | 8    | 7 (14)    | 8    | 42       |
| C    | 9 (18)      | 5    | 6 (12)    | 6    | 41       |

### 4. 可行/期望矩阵

```
        高可行性
           │
    实施区  │  优先区
           │
低期望 ────┼──── 高期望
           │
    放弃区  │  研究区
           │
        低可行性
```

## 组合创新技法

### 1. 形态分析法 (Morphological Analysis)

**步骤**：

1. 确定问题的关键维度
2. 为每个维度列出可能的选项
3. 系统组合不同选项
4. 评估有潜力的组合

**示例 - 设计新型咖啡机**：

| 维度     | 选项 1 | 选项 2 | 选项 3 |
| -------- | ------ | ------ | ------ |
| 加热方式 | 电加热 | 燃气   | 太阳能 |
| 容量     | 单杯   | 多杯   | 商用   |
| 操作     | 手动   | 半自动 | 全自动 |
| 外形     | 传统   | 极简   | 复古   |

可能组合: 太阳能 + 单杯 + 全自动 + 极简 → 户外便携智能咖啡机

### 2. 属性列举法 (Attribute Listing)

**步骤**：

1. 列出产品/服务的所有属性
2. 逐一思考如何改变每个属性
3. 评估改变的价值

**示例 - 改进钢笔**：

| 属性 | 现状 | 变化 | 创新点           |
| ---- | ---- | ---- | ---------------- |
| 墨水 | 液态 | 固态 | 固态墨芯，不漏墨 |
| 笔尖 | 金属 | 陶瓷 | 更耐磨，书写顺滑 |
| 笔身 | 圆柱 | 三角 | 防滚动，更好握持 |

## 突破思维定式

### 假设挑战法

列出所有隐含假设，逐一挑战：

```
问题: 如何缩短产品开发周期？

隐含假设:
1. 必须按顺序开发 → 挑战: 可以并行吗？
2. 必须功能完整 → 挑战: MVP 够用吗？
3. 必须自己开发 → 挑战: 可以外包/购买吗？
4. 必须开发新品 → 挑战: 可以改造现有产品吗？
```

### 极端条件法

在极端条件下思考问题：

- 如果预算无限会怎样？
- 如果预算为零会怎样？
- 如果时间只有一天会怎样？
- 如果时间有十年会怎样？
- 如果用户是儿童会怎样？
- 如果用户是老年人会怎样？

### 反向思维法

1. 定义理想状态
2. 思考如何达到相反状态
3. 将这些方法反转
4. 获得新的解决思路

**示例**：

```
目标: 提高团队协作效率

反向思考 - 如何降低协作效率:
- 不开会 → 定期高效会议
- 信息不共享 → 透明信息平台
- 职责不清 → 明确分工矩阵
- 不给反馈 → 及时反馈机制
```

## 创意评估标准

### NEW 评估法

- **N**ovel (新颖性): 这个想法有多独特？
- **E**ffective (有效性): 能解决问题吗？
- **W**orkable (可行性): 能实现吗？

### 创意温度计

```
🔥 突破性创新 (改变行业规则)
🌡️ 显著改进 (明显优于现有方案)
😊 渐进改进 (小幅提升)
😐 微创新 (换汤不换药)
❄️ 模仿 (无新意)
```
