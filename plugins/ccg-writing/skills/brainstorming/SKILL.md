# Brainstorming - 头脑风暴助手

## 功能概述

通过结构化提问和系统性探索，将模糊的想法转化为清晰的设计方案。

## 头脑风暴框架

### 发散-收敛模型

```
┌─────────────────────────────────────────────────────┐
│                    发散阶段                          │
│  ┌───────────────────────────────────────────────┐  │
│  │   想法生成 → 自由联想 → 疯狂创意 → 跨界借鉴  │  │
│  └───────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────┘
                         ▼
┌─────────────────────────────────────────────────────┐
│                    收敛阶段                          │
│  ┌───────────────────────────────────────────────┐  │
│  │   筛选评估 → 组合优化 → 方案确定 → 行动计划  │  │
│  └───────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────┘
```

### 六顶思考帽

| 帽子 | 颜色 | 思考角度 | 问题示例                     |
| ---- | ---- | -------- | ---------------------------- |
| 白帽 | ⬜   | 客观事实 | 我们知道什么？需要什么数据？ |
| 红帽 | 🟥   | 情感直觉 | 直觉告诉我什么？感觉如何？   |
| 黑帽 | ⬛   | 批判谨慎 | 有什么风险？可能的问题？     |
| 黄帽 | 🟨   | 积极乐观 | 有什么好处？最好的情况？     |
| 绿帽 | 🟩   | 创意创新 | 还有其他方法吗？如何创新？   |
| 蓝帽 | 🟦   | 过程控制 | 下一步是什么？如何组织？     |

## 工作流程

### 1. 问题定义

```markdown
## 问题陈述

### 背景

[描述当前状况和上下文]

### 目标

[期望达成什么？]

### 约束

[有什么限制条件？]

### 成功标准

[如何判断方案成功？]
```

### 2. 想法生成

**自由联想法**

```
核心词: [主题]
    ↓
关联词: [联想1] [联想2] [联想3]
    ↓
延伸: [联想1-1] [联想1-2] [联想2-1] ...
    ↓
灵感: [从延伸中发现的新想法]
```

**SCAMPER 技法**

| 技法                  | 含义 | 应用问题         |
| --------------------- | ---- | ---------------- |
| S - Substitute        | 替代 | 可以用什么替代？ |
| C - Combine           | 结合 | 可以与什么结合？ |
| A - Adapt             | 改造 | 可以怎样调整？   |
| M - Modify            | 修改 | 可以放大/缩小？  |
| P - Put to other uses | 他用 | 还能用在哪里？   |
| E - Eliminate         | 消除 | 可以去掉什么？   |
| R - Reverse           | 逆向 | 可以颠倒/重排？  |

### 3. 方案评估

**评估矩阵**

| 方案 | 可行性 | 成本 | 收益 | 风险 | 总分 |
| ---- | ------ | ---- | ---- | ---- | ---- |
| A    | 8      | 6    | 9    | 7    | 30   |
| B    | 6      | 8    | 7    | 8    | 29   |
| C    | 9      | 5    | 6    | 6    | 26   |

**2x2 优先级矩阵**

```
        高影响
           │
    [待考虑]│[优先实施]
           │
低成本 ────┼──── 高成本
           │
    [暂缓] │[谨慎评估]
           │
        低影响
```

### 4. 方案细化

```markdown
## 方案详细设计

### 核心概念

[一句话描述方案核心]

### 关键特性

1. [特性1]: [描述]
2. [特性2]: [描述]
3. [特性3]: [描述]

### 实施步骤

1. [步骤1]
2. [步骤2]
3. [步骤3]

### 所需资源

- 人力: [需求]
- 技术: [需求]
- 资金: [估算]

### 风险与应对

| 风险 | 概率 | 影响 | 应对措施 |
| ---- | ---- | ---- | -------- |
|      |      |      |          |

### 成功指标

- [指标1]: [目标值]
- [指标2]: [目标值]
```

## 常用提问清单

### 探索问题

- 这个问题的本质是什么？
- 谁受到这个问题的影响？
- 问题存在多久了？
- 之前尝试过什么解决方案？
- 为什么之前的方案没有成功？

### 激发创意

- 如果没有任何限制，理想的解决方案是什么？
- 竞争对手/其他行业是如何解决类似问题的？
- 如果完全相反会怎样？
- 可以从哪里借鉴灵感？
- 最简单的解决方案是什么？

### 评估方案

- 这个方案能解决根本问题吗？
- 实施这个方案需要什么条件？
- 有什么潜在的负面影响？
- 如何衡量方案的成功？
- 如果失败了，后备方案是什么？
