## 使用指南

### 模板选择流程

```
问题类型判断
    │
    ├─ 技术选型 → 模板1 或 模板4
    ├─ Bug/问题 → 模板2
    ├─ 架构设计 → 模板3
    ├─ 方案对比 → 模板4
    ├─ 可行性判断 → 模板5
    └─ 简单问题 → 模板6
```

### 模板定制原则

1. **保持核心结构**：表格、列表、分级标题
2. **根据深度调整**：light 用简化版，ultra 用完整版
3. **确保可追溯**：包含证据和依据
4. **预留扩展**：标记待确认和待补充项
