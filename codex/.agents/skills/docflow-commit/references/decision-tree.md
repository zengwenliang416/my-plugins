# 决策树：docflow-commit

## 提交流程判断

[用户提出提交请求]
├── 否：返回“不触发”，建议使用对应技能。
└── 是：继续
    └── [是否存在代码变更]
        ├── 否：返回“无可提交变更”。
        └── 是：继续
            └── [是否存在 staged 变更]
                ├── 否：询问是否先暂存。
                │   ├── 取消：结束。
                │   └── 继续：生成候选 message。
                └── 是：直接生成候选 message。
                    └── [用户是否确认提交]
                        ├── 否：返回候选 message 供后续复用。
                        └── 是：执行 commit 并返回结果。
